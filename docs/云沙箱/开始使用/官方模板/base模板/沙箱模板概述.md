# 沙箱模板概述

沙箱模板定义了沙箱实例的基础运行环境配置。每个沙箱实例在创建时都需要关联一个模板，模板决定了沙箱的容器镜像、实例规格（CPU、内存、磁盘）、启动命令等核心参数。通过模板机制，您可以将常用的环境配置固化为模板，在创建沙箱实例时直接引用，避免重复配置。

## 模板与实例的关系

沙箱模板和沙箱实例是"配置定义"与"运行实体"的关系：

| **概念** | **说明** |
| --- | --- |
| 沙箱模板 | 一组沙箱基础配置的集合，包括容器镜像、实例规格、环境变量等。模板创建后可反复使用，本身不消耗计算资源。 |
| 沙箱实例 | 基于模板创建的实际运行环境。每个实例独占一个函数实例（SESSION_EXCLUSIVE 隔离模式），拥有独立的 CPU、内存、磁盘和网络空间。 |

一个模板可以创建多个沙箱实例，但每个实例在任意时刻只服务于一个会话。

## 预置模板类型

云沙箱提供三种预置沙箱镜像模板，覆盖不同的业务场景：

| **模板类型** | **模板标识** | **默认规格** | **典型场景** |
| --- | --- | --- | --- |
| Code Interpreter | `CodeInterpreter` | 2 vCPU / 4096 MB / 512 MB 磁盘 | AI Agent 代码执行、数据分析、文件处理 |
| Browser Tool | `Browser` | 2 vCPU / 4096 MB / 10240 MB 磁盘 | 浏览器自动化、网页抓取、截图、UI 测试 |
| All-in-One | `AllInOne` | 4 vCPU / 8192 MB / 10240 MB 磁盘 | 需要同时使用代码执行和浏览器自动化的复合场景 |

此外，您还可以通过**自定义模板**使用自己的容器镜像，预装业务所需的依赖、工具和证书，构建标准化的专属沙箱环境。

## 如何选择模板

```
需要浏览器自动化（网页操作、截图、数据采集）？
├── 是 → 还需要代码执行（Python/JS/Shell）？
│       ├── 是 → 选择 All-in-One 模板
│       └── 否 → 选择 Browser 沙箱
└── 否 → 选择 Code Interpreter 沙箱
```

如果您有预装特定依赖或工具链的需求（如特定的 Python 库、业务证书、自定义运行时等），可以选择自定义模板，通过自定义容器镜像构建专属环境。

## 模板配置参数

无论选择哪种模板类型，创建模板时均可配置以下参数：

| **配置项** | **说明** |
| --- | --- |
| CPU | 沙箱实例的 CPU 核数，最低 2 vCPU |
| 内存 | 沙箱实例的内存大小，最低 4096 MB |
| 磁盘大小 | 临时存储空间，可选 512 MB 或 10240 MB |
| 环境变量 | 运行时环境变量，用于传递配置信息 |
| 超时时间 | 单次请求的超时时间，取值范围 1~86400 秒 |
| 日志配置 | 日志服务配置，用于采集沙箱运行日志 |
| 网络配置 | VPC 网络配置，用于访问私有网络资源（挂载 NAS 时必须） |
| 存储挂载 | 支持 NAS、OSS、PolarFS 动态挂载 |

## 使用方式

您可以通过以下方式管理沙箱模板：

| **方式** | **说明** |
| --- | --- |
| 函数计算控制台 | 在创建沙箱函数时选择预置镜像或自定义镜像，系统自动创建对应模板 |
| E2B SDK | 通过 E2B 兼容的 Python/TypeScript SDK 进行模板管理和沙箱实例创建 |

详细的模板创建步骤，请参见[创建沙箱函数（模板）](../../功能说明/创建沙箱函数（模板）.md)。
