# 使用 AIO Sandbox (邀测)

> 该部分功能正在内部邀请测试中，内容可能会根据测试情况进行修改，请注意。

AIO Sandbox 面向“浏览器操作 + 代码执行 + 文件处理”连续发生的 Agent 任务。开发者可以优先使用云沙箱内置 `all in one` 模板（同时包含浏览器、Playwright/Puppeteer、Python/Node.js 和常用数据处理依赖），即可在同一个沙箱中完成网页访问、截图、下载、数据清洗、代码计算和报告生成，减少浏览器环境与代码执行环境之间的数据搬运。

> 官方功能文档中的模板名称为 `all in one`。创建前可用 `e2b template list` 确认目标账号和地域中实际可用的模板名；如果 CLI/控制台显示的是自定义别名，请将示例中的 `all-in-one-template` 替换为该名称。只有内置模板无法满足企业依赖、证书、字体或运行时版本要求时，再基于 `browser` 或 `all in one` 镜像构建自定义模板。

这个实践适合网页数据采集后分析、自动化测试后生成报告、内容生产 Agent、运营后台巡检和多工具 Agent。相比只使用 Browser Use Sandbox，AIO Sandbox 更适合需要把浏览器产物立即交给 Python/Node 脚本继续处理的任务。

## 业务场景

- 从动态网页采集数据，立即运行 Python 脚本清洗、汇总并导出文件。
- 在浏览器中完成操作后，对下载文件做解析、转码或报表生成。
- 自动化测试中同时运行浏览器 E2E 和后端脚本。
- 为内容生产 Agent 生成截图、PDF、表格和归档文件。

## 推荐流程

1. 确认目标账号已开放 `all in one` 模板，或已发布等价自定义模板别名（可用 `e2b template list` 校验）。
2. 使用 `all in one` 模板创建云沙箱。
3. 浏览器阶段访问网页、交互、截图或下载文件。
4. 代码阶段读取浏览器产物，完成解析、清洗、计算和报告生成。
5. 审计阶段保存 URL、动作摘要、截图路径、脚本版本和输出结果。
6. 清理阶段释放沙箱，删除任务级凭证和临时文件。

## 示例代码

以下示例假设 `all in one` 模板已预装 Playwright、浏览器和 Python（依赖在模板构建期预装，不要在每次创建沙箱后现装），用于先采集网页信息，再生成分析结果。

> E2B SDK 默认在命令非零退出时抛出 `CommandExitError`（含 `exitCode`/`stdout`/`stderr`），不会返回到 `exitCode` 分支。示例使用 try/catch 捕获并归一化错误。若 `apiUrl`/`domain` 指向自建或本地网关（自签证书），需在 Node 侧设置 `NODE_EXTRA_CA_CERTS`（Python 侧 `SSL_CERT_FILE`/`REQUESTS_CA_BUNDLE`）信任该 CA；生产官方域名无此问题。

```javascript
import { Sandbox } from "e2b";

const template = process.env.AIO_TEMPLATE || "all-in-one-template";

const sandbox = await Sandbox.create(template, {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  envs: { TASK_TYPE: "aio-browser-code" },
  metadata: { scenario: "aio-browser-code" },
  timeoutMs: 300_000,
});

try {
  await sandbox.files.makeDir("/tmp/aio-task");
  await sandbox.files.write(
    "/tmp/aio-task/collect.mjs",
    `import fs from "node:fs/promises";
import { chromium } from "playwright";

const browser = await chromium.launch({ headless: true });
const page = await browser.newPage({ viewport: { width: 1365, height: 768 } });
await page.goto("https://example.com", { waitUntil: "networkidle" });
await page.screenshot({ path: "page.png", fullPage: true });

const rows = await page.evaluate(() => [{
  url: location.href,
  title: document.title,
  textLength: document.body.innerText.length,
}]);

await fs.writeFile("browser-result.json", JSON.stringify(rows, null, 2));
await browser.close();
`,
  );
  await sandbox.files.write(
    "/tmp/aio-task/analyze.py",
    `import json
from pathlib import Path

rows = json.loads(Path("browser-result.json").read_text())
summary = {
    "page_count": len(rows),
    "titles": [row["title"] for row in rows],
    "total_text_length": sum(row["textLength"] for row in rows),
    "artifacts": ["page.png", "browser-result.json"],
}
Path("summary.json").write_text(json.dumps(summary, ensure_ascii=False, indent=2))
print(json.dumps(summary, ensure_ascii=False))
`,
  );

  const collect = await runCmd(sandbox, "node collect.mjs", "/tmp/aio-task", 180_000);
  const analyze = await runCmd(sandbox, "python3 analyze.py", "/tmp/aio-task", 60_000);

  console.log(JSON.parse(analyze.stdout));
} finally {
  await sandbox.kill();
}

// 运行命令并在非零退出时抛出归一化错误。
// E2B SDK 默认对非零退出抛 CommandExitError（含 exitCode/stdout/stderr），这里用 try/catch 统一捕获。
async function runCmd(sandbox, cmd, cwd, timeoutMs) {
  try {
    return await sandbox.commands.run(cmd, { cwd, timeoutMs });
  } catch (e) {
    throw new Error(e.stderr || e.stdout || String(e));
  }
}
```

## 上线建议

- 将浏览器阶段和代码阶段拆开记录，分别保存输入、输出、日志和错误。
- 下载文件先做类型、大小和路径校验，再交给代码阶段处理。
- 网页内容和下载文件都可能携带 prompt injection，不要把网页文本直接作为系统指令。
- 浏览器账号、Cookie、模型 Key 和业务凭证运行时注入，任务结束后销毁。
- 对高成本任务设置超时、文件大小上限和最大页面数，避免 Agent 无限制爬取或计算。
