# 使用 Computer Use Sandbox（邀测）

> 该部分功能正在内部邀请测试中，内容可能会根据测试情况进行修改，请注意。

Computer Use Sandbox 面向需要“看屏幕、动鼠标、敲键盘”的 Agent。它在云沙箱中运行一套完整的 Linux 桌面环境（基于 [Xfce](https://www.xfce.org/) 与 Xvfb 虚拟显示），Agent 通过截图感知界面，通过鼠标、键盘操作任意 GUI 应用，并可实时查看整个桌面。与只暴露浏览器 CDP 的 [Browser Use Sandbox](02.使用%20Browser%20Use%20Sandbox.md) 不同，Computer Use 面向的是**整台虚拟电脑**：浏览器、文件管理器、编辑器、终端等桌面程序都能驱动。

如果任务只需要访问网页、点击、填表、截图或抓取动态页面，优先使用 [Browser Use Sandbox](02.使用%20Browser%20Use%20Sandbox.md)——它更轻量，且通过 CDP 精确控制 DOM。只有当任务必须操作浏览器以外的桌面应用，或必须以“像素坐标 + 截图”的视觉方式驱动界面（例如接入视觉大模型的 Computer Use 智能体）时，才选择 Computer Use Sandbox。

## 适用场景

| 场景 | 说明 |
| --- | --- |
| 视觉驱动的 Computer Use Agent | 让视觉大模型基于截图输出鼠标坐标和键盘动作，闭环操作桌面 |
| 桌面软件自动化 | 驱动浏览器以外的 GUI 应用，如编辑器、办公软件、桌面客户端 |
| 端到端 GUI 测试 | 在隔离桌面中回归验证图形界面，避免污染本地或 CI 环境 |
| 演示与远程观察 | 通过实时流查看 Agent 的每一步桌面操作，便于演示、调试和审计 |

## 与 Browser Use Sandbox 的区别

| 对比项 | Computer Use Sandbox | Browser Use Sandbox |
| --- | --- | --- |
| 环境形态 | 完整 Linux 桌面（Xfce + Xvfb） | 无桌面，仅浏览器实例 |
| 操作对象 | 任意桌面 GUI 应用 | 仅浏览器页面 |
| 控制方式 | 截图 + 鼠标坐标 + 键盘（视觉驱动） | CDP over WebSocket（DOM 精确控制） |
| SDK | `e2b-desktop` / `@e2b/desktop` | `e2b` / `@e2b/code-interpreter` + Puppeteer/Playwright |
| 实时查看 | SDK 内置 `stream`，返回可分享 URL | `wss://<sandbox-host>/ws/livestream`（noVNC） |
| 适合任务 | 桌面级、视觉级操作 | 网页数据采集、表单、轻量 E2E |

## 准备模板

Computer Use Sandbox 与 [browser 模板](../04.功能说明/02.模板/07.示例/01.browser-模板.md)一样，不是开通即用的内置模板，需要先从桌面预置镜像构建一个带名称的自定义模板，再用该模板创建沙箱。构建方式与 browser 模板一致，仅需替换镜像与模板名称：

```python
"""Computer Use 模板构建示例。"""

import os

from dotenv import load_dotenv
from e2b import Template, default_build_logger

load_dotenv()

# SDK 自动读取 E2B_API_KEY / E2B_API_URL / E2B_DOMAIN 环境变量。
# 邀测期间请从平台侧获取当前地域的桌面预置镜像地址，通过环境变量注入，不要硬编码。
FROM_IMAGE = os.environ["E2B_DESKTOP_IMAGE"]

build = Template.build(
    Template().from_image(FROM_IMAGE),
    name="my-desktop-template",
    cpu_count=4,
    memory_mb=8192,
    on_build_logs=default_build_logger(),
)
print(f"template_id: {build.template_id}")
```

邀测镜像地址与 tag 可能随版本和地域调整，请从邀测联系人处获取。上例使用 4 vCPU、8192 MB 内存作为推荐起点，不代表平台允许的最低规格。本文用环境变量 `E2B_DESKTOP_IMAGE` 表示镜像地址、`E2B_DESKTOP_TEMPLATE` 表示构建后的模板名称，请勿在代码中写死镜像。

## 准备本地环境

Computer Use Sandbox 使用独立的桌面 SDK，无需在本地安装浏览器或桌面环境，桌面运行在云沙箱内。

**Python**（`e2b-desktop` 2.4.1）：

```bash
uv venv .venv --python 3.12
source .venv/bin/activate
uv pip install e2b-desktop==2.4.1 python-dotenv
```

**Node.js**（`@e2b/desktop` 2.3.1）：使用如下 `package.json`，然后执行 `npm install`。

```json
{
  "name": "computer-use-sandbox-demo",
  "version": "1.0.0",
  "type": "module",
  "dependencies": {
    "@e2b/desktop": "2.3.1"
  }
}
```

> **说明**：`E2B_API_KEY`、`E2B_API_URL`、`E2B_DOMAIN` 需在运行前通过环境变量（如 `.env` 或 `export`）配置为对应地域的接入地址，SDK 会自动读取，无需在方法调用中显式传入。密钥应放在 `.env` 并通过 `.gitignore` 排除，不要提交到版本库。

## Python 示例

示例流程：创建桌面沙箱 → 启动 Chrome → 截图 → 鼠标与键盘操作 → 开启实时流 → `try/finally` 中销毁沙箱。

```python
"""Computer Use Sandbox 示例：桌面创建 -> 启动 Chrome -> 截图 -> 鼠标键盘 -> 实时流。"""

import os

from dotenv import load_dotenv
from e2b_desktop import Sandbox

load_dotenv()

# SDK 自动读取 E2B_API_KEY / E2B_API_URL / E2B_DOMAIN 环境变量。
TEMPLATE = os.environ.get("E2B_DESKTOP_TEMPLATE", "my-desktop-template")

desktop = None
try:
    # 创建桌面沙箱，timeout 单位为秒，需覆盖桌面冷启动与任务全过程
    desktop = Sandbox.create(template=TEMPLATE, timeout=600)

    # 启动 Chrome，并等待窗口就绪（wait 单位为毫秒）
    desktop.launch("google-chrome", "https://example.com")
    desktop.wait(10000)

    # 截图：返回 PNG 字节，可交给视觉模型或落盘
    image = desktop.screenshot()
    with open("desktop.png", "wb") as f:
        f.write(image)

    # 鼠标操作：移动、单击、双击、向下滚动 5 格
    desktop.move_mouse(640, 400)
    desktop.left_click()
    desktop.double_click()
    desktop.scroll("down", 5)

    # 键盘操作：输入文本与按键组合
    desktop.write("hello, computer use")
    desktop.press(["ctrl", "a"])
    desktop.press("enter")

    # 开启实时流（带鉴权），获取可分享的观察 URL
    # 同一时刻只能有一个流，切换前需先 stop
    desktop.stream.start(require_auth=True)
    auth_key = desktop.stream.get_auth_key()
    print("Stream URL:", desktop.stream.get_url(auth_key=auth_key))

    # ... 在此执行 Agent 的截图-决策-操作循环 ...

    desktop.stream.stop()
finally:
    # 任务结束务必销毁沙箱，释放资源并清理临时状态
    if desktop is not None:
        desktop.kill()
```

## TypeScript 示例

```typescript
import { writeFileSync } from "node:fs";
import { Sandbox } from "@e2b/desktop";

// SDK 自动读取 E2B_API_KEY / E2B_API_URL / E2B_DOMAIN 环境变量。
const TEMPLATE = process.env.E2B_DESKTOP_TEMPLATE ?? "my-desktop-template";

let desktop: Sandbox | undefined;
try {
  // 创建桌面沙箱，timeoutMs 单位为毫秒，需覆盖桌面冷启动与任务全过程
  desktop = await Sandbox.create(TEMPLATE, { timeoutMs: 600_000 });

  // 启动 Chrome，并等待窗口就绪（wait 单位为毫秒）
  await desktop.launch("google-chrome", "https://example.com");
  await desktop.wait(10_000);

  // 截图：返回 PNG 字节，可交给视觉模型或落盘
  const image = await desktop.screenshot();
  writeFileSync("desktop.png", image);

  // 鼠标操作：移动、单击、双击、向下滚动 5 格
  await desktop.moveMouse(640, 400);
  await desktop.leftClick();
  await desktop.doubleClick();
  await desktop.scroll("down", 5);

  // 键盘操作：输入文本与按键组合
  await desktop.write("hello, computer use");
  await desktop.press(["ctrl", "a"]);
  await desktop.press("enter");

  // 开启实时流（带鉴权），获取可分享的观察 URL
  // 同一时刻只能有一个流，切换前需先 stop
  await desktop.stream.start({ requireAuth: true });
  const authKey = desktop.stream.getAuthKey();
  console.log("Stream URL:", desktop.stream.getUrl({ authKey }));

  // ... 在此执行 Agent 的截图-决策-操作循环 ...

  await desktop.stream.stop();
} finally {
  // 任务结束务必销毁沙箱，释放资源并清理临时状态
  await desktop?.kill();
}
```

## 常用能力速查

| 能力 | Python | TypeScript |
| --- | --- | --- |
| 创建沙箱 | `Sandbox.create(template=..., timeout=...)` | `Sandbox.create(template, { timeoutMs })` |
| 启动应用 | `desktop.launch("google-chrome", url)` | `desktop.launch("google-chrome", url)` |
| 等待 | `desktop.wait(10000)` | `desktop.wait(10000)` |
| 截图 | `desktop.screenshot()` → PNG 字节 | `desktop.screenshot()` → PNG 字节 |
| 移动鼠标 | `desktop.move_mouse(x, y)` | `desktop.moveMouse(x, y)` |
| 单击 / 双击 / 右键 | `desktop.left_click()` / `double_click()` / `right_click()` | `desktop.leftClick()` / `doubleClick()` / `rightClick()` |
| 滚动 | `desktop.scroll("down", amount)` | `desktop.scroll("down", amount)` |
| 拖拽 | `desktop.drag((x1, y1), (x2, y2))` | `desktop.drag([x1, y1], [x2, y2])` |
| 输入文本 | `desktop.write("...")` | `desktop.write("...")` |
| 按键 / 组合键 | `desktop.press("enter")` / `press(["ctrl", "c"])` | `desktop.press("enter")` / `press(["ctrl", "c"])` |
| 当前窗口 ID | `desktop.get_current_window_id()` | `desktop.getCurrentWindowId()` |
| 开启 / 停止实时流 | `desktop.stream.start()` / `stream.stop()` | `desktop.stream.start()` / `stream.stop()` |
| 获取流地址 | `desktop.stream.get_url(auth_key=...)` | `desktop.stream.getUrl({ authKey })` |
| 销毁沙箱 | `desktop.kill()` | `desktop.kill()` |

`stream.get_url()` 返回 noVNC 实时流地址。通过函数计算公网网关访问 `/vnc.html` 和 `/websockify` 时，还需要沙箱的 `X-Access-Token`；`require_auth=True` 生成的 `auth_key` 是 VNC 流自身的访问口令，两者是独立的两层鉴权。普通浏览器无法为页面和 WebSocket 任意附加请求头，面向最终用户提供实时查看时，建议由业务后端通过受控反向代理注入 `X-Access-Token`，不要把沙箱 Token 暴露到前端或公开日志。

## 实时流与只读观察

- 开启实时流后，`stream.get_url()` 返回观察地址；`require_auth=True` 会生成 `auth_key`。在函数计算公网网关场景，需先通过受控代理补充 `X-Access-Token`，再向浏览器提供观察地址。
- 若只需展示、不允许他人操作，可开启只读模式：Python `stream.get_url(auth_key=..., view_only=True)`，TypeScript `stream.getUrl({ authKey, viewOnly: true })`。
- 同一时刻只能有一个流。若要从整桌面切换到某个窗口，或切换观察对象，需先 `stream.stop()` 再重新 `stream.start()`。
- 只流某个应用窗口时，先用 `get_current_window_id()` / `get_application_windows(name)` 拿到窗口 ID，再传入 `stream.start(window_id=...)`；该应用必须已经打开，且窗口关闭后流会随之关闭。

## 安全与上线建议

- **凭证隔离**：模型 Key、账号密码、业务 Token 通过运行时环境变量注入，不要写入模板镜像；密钥文件放入 `.env` 并用 `.gitignore` 排除。
- **流访问控制**：实时流始终开启 `require_auth=True`，只读演示场景使用 `view_only`；不要把带 `auth_key` 的 URL 记录到公开日志。
- **防提示注入**：截图中的网页或应用内容可能包含诱导指令。不要让 Agent 未经校验地把屏幕文字当作系统指令执行，对敏感操作（提交、删除、支付）设置人工确认。
- **资源与生命周期**：为 `timeout` / `timeoutMs` 设置合理上限，任务无论成功或异常都在 `finally` 中 `kill()`，避免桌面沙箱长时间占用资源。
- **动作审计**：记录每一步的截图、鼠标坐标、按键序列和目标窗口，便于复现问题和追踪 Agent 行为。
- **最小权限**：限制沙箱可访问的域名、可启动的应用和可写目录，缩小视觉驱动操作的影响面。
