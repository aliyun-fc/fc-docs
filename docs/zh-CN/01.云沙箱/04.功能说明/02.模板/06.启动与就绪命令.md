# 启动与就绪命令

E2B 模板支持在模板配置中声明启动命令和就绪检查。该能力当前仅在 micro-sandbox 运行时下生效；标准模板（如 `base`、`code-interpreter-v1`）在沙箱创建阶段不执行模板级启动命令，应在创建沙箱后通过 `commands.run` 启动服务。

云沙箱当前应优先使用沙箱创建后的命令执行方式启动业务进程：

```typescript
const server = await sandbox.commands.run("python3 -m http.server 8000", {
  background: true,
});

const host = sandbox.getHost(8000);
console.log(`https://${host}`);
```

Python 示例：

```python
server = sandbox.commands.run(
    "python3 -m http.server 8000",
    background=True,
)

host = sandbox.get_host(8000)
print(f"https://{host}")
```

## 使用建议

如果服务启动步骤属于每次沙箱都必须执行的初始化逻辑，可以先把依赖安装前置到模板，再在创建沙箱后用后台命令启动服务。标准模板当前不提供模板级启动命令，应在每次创建沙箱后用 `commands.run(background=True)` 启动所需服务。
