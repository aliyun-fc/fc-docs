# 模板名称与版本

模板名称标识运行环境；标签为模板的某次构建标记阶段或版本（如 `prod`、`staging`、`v1`），便于灰度和回滚。

开始前，请先完成[快速开始](01.快速开始.md)中的连接配置。

## 模板名称

名称建议包含业务场景、语言或基础镜像、关键依赖版本和日期，便于识别和分组：

```text
agent-python313-20260704
code-review-node22-20260704
data-analysis-py313-v1
```

创建沙箱时用名称引用模板：

```typescript
const sandbox = await Sandbox.create("agent-python313-20260704");
```

Python 示例：

```python
sandbox = Sandbox.create(template="agent-python313-20260704")
```

## 标签

### 分配标签

`assignTags` 第一个参数使用 `name:tag` 格式指定从哪个构建打标签；不带 `:tag` 时默认指向 `default` 构建。

```typescript
import { Template } from "e2b";

// 不指定源标签时，为 default 构建分配标签
await Template.assignTags("agent-python313-20260704", ["staging", "v1"]);
```

Python 示例：

```python
from e2b import Template

Template.assign_tags("agent-python313-20260704", ["staging", "v1"])
```

### 查询标签

分配与移除标签按模板名称操作；查询标签时，TypeScript 用模板名称即可返回全部标签，Python 需使用模板 ID（例如 `Template.build` 返回的 `build.template_id`）。

```typescript
const tags = await Template.getTags("agent-python313-20260704");
console.log(tags);
// 输出: [{ tag: 'default', buildId: '...', createdAt: '...' },
//       { tag: 'staging', buildId: '...', createdAt: '...' }, ...]
```

Python 示例：

```python
tags = Template.get_tags("<template-id>")
print(tags)
# 输出: [TemplateTag(tag='default', build_id='...', created_at='...'), ...]
```

### 移除标签

```typescript
await Template.removeTags("agent-python313-20260704", "staging");
```

Python 示例：

```python
Template.remove_tags("agent-python313-20260704", "staging")
```

### 限制说明

- 每个模板最多 20 个标签。
- 不能删除 `default` 标签（返回 400 错误）。
- 重复分配同一标签、删除不存在的标签都是幂等操作。

## 使用建议

发布新版本时，先验证新模板或标签，再切换业务引用；复杂的发布规则仍由业务系统或发布系统管理。
