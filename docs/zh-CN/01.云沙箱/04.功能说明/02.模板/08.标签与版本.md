# 标签与版本

标签用于给模板的构建增加可读的版本或环境标记，例如 `prod`、`staging`、`v1`。适合做灰度、回滚和模板分组。

## 前置：连接配置

E2B SDK 会自动读取 `E2B_API_KEY`、`E2B_API_URL`、`E2B_DOMAIN` 环境变量，下方示例均不再逐调用显式传入这些参数。开始前请先设置：

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.cn-hangzhou.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="cn-hangzhou.e2b.fc.aliyuncs.com"
```

## 分配标签

`assignTags` 的第一个参数使用 `name:tag` 格式，指定从哪个构建打标签。不传 `:tag` 时默认指向 `default` 构建。

```typescript
import { Template } from "e2b";

// 为默认构建分配标签
await Template.assignTags("my-template", ["staging", "v1"]);

// 为指定构建分配标签
await Template.assignTags("my-template:default", ["staging", "v1"]);
```

## 查询标签

```typescript
const tags = await Template.getTags("my-template");

console.log(tags);
// 输出: [{ tag: 'default', buildId: '...', createdAt: '...' }, ...]
```

Python 示例：

```python
from e2b import Template

# 为默认构建分配标签
Template.assign_tags("my-template", ["staging", "v1"])

# 查询模板的所有标签
tags = Template.get_tags("my-template")

print(tags)
```

## 移除标签

```typescript
await Template.removeTags("my-template", "staging");
```

Python 示例：

```python
Template.remove_tags("my-template", "staging")
```

**限制说明：**

- 每个模板最多 20 个标签
- 不能删除 `default` 标签（会返回 400 错误）
- 重复分配同一标签是幂等操作
- 删除不存在的标签也是幂等操作

## 使用建议

标签适合表达“阶段”和“版本”，模板名称适合表达“运行环境”。不要把复杂发布规则完全塞进标签，生产切换仍应由业务系统或发布系统控制。
