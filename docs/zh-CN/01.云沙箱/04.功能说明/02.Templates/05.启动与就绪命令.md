# 启动与就绪命令

E2B 模板支持在模板中配置启动命令和就绪检查，用于让沙箱创建后自动启动服务并等待服务可用。

云沙箱当前应优先使用沙箱创建后的命令执行方式启动业务进程：

**TypeScript 示例**：

```typescript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create({
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  timeoutMs: 300_000,
});

try {
  const server = await sandbox.commands.run("python3 -m http.server 8000", {
    background: true,
  });

  const host = sandbox.getHost(8000);
  console.log(`https://${host}`);
  
  console.log("服务存活 15s, 可现在访问上面的地址...");
  await new Promise((r) => setTimeout(r, 15_000));
} finally {
  await sandbox.kill();
  console.log("沙箱已销毁");
}
```

**Python 示例**：

```python
import os
import time

from e2b import Sandbox

sandbox = Sandbox.create(
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
    timeout=300,
)

try:
    server = sandbox.commands.run(
        "python3 -m http.server 8000",
        background=True,
    )

    host = sandbox.get_host(8000)
    print(f"https://{host}")
    
    print("服务存活 15s, 可现在访问上面的地址...")
    time.sleep(15)
finally:
    sandbox.kill()
    print("沙箱已销毁")

```

## 使用建议

如果服务启动步骤属于每次沙箱都必须执行的初始化逻辑，可以先把依赖安装前置到模板，再在创建沙箱后用后台命令启动服务。模板级启动与就绪命令是否完全兼容，需要以实际 SDK 与构建结果验证为准。
