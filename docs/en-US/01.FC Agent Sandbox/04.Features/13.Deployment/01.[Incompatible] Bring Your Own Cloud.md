# [Incompatible] Bring Your Own Cloud

Bring Your Own Cloud (BYOC) is a deployment model on the E2B platform where the E2B control plane or data plane is deployed into the user's own cloud account, private cloud, or Kubernetes cluster.

FC Agent Sandbox does not support E2B BYOC. FC Agent Sandbox is a cloud service provided by Alibaba Cloud Function Compute. Developers integrate through the FC Agent Sandbox endpoint, API Key, templates, and Sandbox management capabilities. You do not need to — and cannot — deploy the FC Agent Sandbox control plane yourself.

## When you may encounter this difference

When migrating from E2B, if your application or operations documentation mentions any of the following, you are likely using E2B BYOC:

- Deploying the E2B control plane in the user's own cloud account.
- Running E2B infrastructure in a self-managed Kubernetes cluster.
- Managing E2B data plane, scheduling, image repositories, network ingress, or cluster upgrades yourself.
- Deploying the platform to a specific cloud environment through E2B Deployment configuration.

These capabilities are not within the FC Agent Sandbox compatibility scope. Do not apply E2B BYOC deployment steps directly to FC Agent Sandbox when migrating.

## Common alternative paths

Many requirements that "look like BYOC" are actually about letting a Sandbox access the enterprise intranet, use a fixed domain name, mount data, or connect to observability capabilities. FC Agent Sandbox handles these scenarios with FC Extensions:

- Let a Sandbox access databases, internal APIs, or image repositories within a VPC: use [VPC Network Configuration](../07.FC%20Extensions/02.VPC%20Network%20Configuration.md).
- Use a fixed domain name and certificate to access the FC Agent Sandbox API or services inside a Sandbox: use [Custom Domains](../07.FC%20Extensions/03.Custom%20Domains.md).
- Mount OSS data into a Sandbox: use [Dynamic OSS Mounts](../07.FC%20Extensions/04.Dynamic%20OSS%20Mounts.md).
- View runtime logs, resource metrics, or integrate with log services: use [Monitoring and Logs](../07.FC%20Extensions/05.Monitoring%20and%20Logs.md).

These capabilities are cloud extensions of FC Agent Sandbox and are not part of E2B Deployment / BYOC.

## Migration recommendations

When migrating E2B applications that depend on BYOC, first break down the actual requirements, then choose the path:

- Need private network access, domains, object storage, or log monitoring: migrate step by step using FC Extensions.
- Need data residency or compliance isolation: choose the appropriate Alibaba Cloud region and configure VPC, permissions, logging, and storage policies.
- Need to deploy the entire sandbox control plane to your own cloud account or private cluster: this is not within the current FC Agent Sandbox compatibility scope. Contact product support to confirm whether an enterprise delivery solution is available.

Application code typically does not need to be aware of BYOC. Keep the E2B SDK calling conventions, replace with FC Agent Sandbox connection parameters, and configure cloud extensions as needed.
