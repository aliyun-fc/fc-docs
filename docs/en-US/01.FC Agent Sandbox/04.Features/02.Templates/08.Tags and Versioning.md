# Tags and Versioning

Tags add readable version or environment markers to a template, such as `prod`, `staging`, or `v1`. They are useful for canary rollout, rollback, and template grouping.

## Connection configuration

The E2B SDK automatically reads `E2B_API_KEY`, `E2B_API_URL`, and `E2B_DOMAIN`. Set them before running the examples below:

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.cn-hangzhou.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="cn-hangzhou.e2b.fc.aliyuncs.com"
```

## Assign tags

The first parameter of `assignTags` uses the `name:tag` format to specify which build to tag. When `:tag` is not passed, it defaults to the `default` build.

```typescript
import { Template } from "e2b";

// Tag the default build
await Template.assignTags("agent-python313-20260704", ["staging", "v1"]);

// Tag a specified build
await Template.assignTags("agent-python313-20260704:default", ["staging", "v1"]);
```

## Query tags

```typescript
const tags = await Template.getTags("agent-python313-20260704");

console.log(tags);
```

Python example:

```python
from e2b import Template

Template.assign_tags(
    "agent-python313-20260704",
    ["staging", "v1"],
)

tags = Template.get_tags("agent-python313-20260704")

print(tags)
```

## Remove tags

```typescript
await Template.removeTags("agent-python313-20260704", "staging");
```

Python example:

```python
Template.remove_tags(
    "agent-python313-20260704",
    "staging",
)
```

**Limitations:**

- Each template supports at most 20 tags.
- The `default` tag cannot be deleted (returns a 400 error).
- Assigning the same tag repeatedly is idempotent.
- Deleting a tag that does not exist is also idempotent.

## Recommendations

Use tags for stage and version, and use template names for the runtime environment itself. Do not force complex release logic entirely into tags. Production cutovers should still be controlled by the application or release system.
