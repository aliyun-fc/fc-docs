# Start and Ready Commands

E2B templates support startup commands and readiness checks in template configuration. In FC Agent Sandbox, this capability currently takes effect only for the `micro-sandbox` runtime. Standard templates such as `base` and `code-interpreter-v1` do not run template-level startup commands when a sandbox is created. Start services with `commands.run` after creating the sandbox.

FC Agent Sandbox should currently prefer starting application processes by running commands after the sandbox is created:

```typescript
const server = await sandbox.commands.run("python3 -m http.server 8000", {
  background: true,
});

const host = sandbox.getHost(8000);
console.log(`https://${host}`);
```

Python example:

```python
server = sandbox.commands.run(
    "python3 -m http.server 8000",
    background=True,
)

host = sandbox.get_host(8000)
print(f"https://{host}")
```

## Recommendations

If the service startup step is required every time a sandbox is created, install the dependencies in the template first, then start the service with a background command after sandbox creation. For standard templates, start the required service with `commands.run(..., { background: true })` each time you create a sandbox.
